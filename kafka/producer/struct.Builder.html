fn with_required_acks(self, required_acks: i16) -> Self

Sets how many acknowledgements the kafka brokers should receive before responding to sent messages. If it is 0 the servers will not send any response. If it is 1, the server will wait the data is written to the local server log before sending a replying. If it is -1 the servers will block until the messages are committed by all in sync replicas before replaying. For any number > 1 the servers will block waiting for this number of acknowledgements to occur (but the servers will never wait for more acknowledgements than there are in-sync replicas).
